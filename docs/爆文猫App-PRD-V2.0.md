# 爆文猫App PRD V2.0

## 1. 文档信息
- 产品名称：爆文猫App
- 文档版本：V2.0（可评审版）
- 目标读者：产品、设计、研发、测试、运营

## 2. 产品目标与范围（MVP）

### 2.1 产品目标
打造“轻量化AI互动叙事平台”，优先验证“读得爽 + 互动强 + 创作快”的核心价值。

### 2.2 MVP核心闭环
发现内容 -> 沉浸阅读 -> 分支互动 -> 回跳探索 -> 章节解锁 -> 轻创作发布。

### 2.4 MVP不做
- 抽卡付费机制——邂逅模块删掉
- 对话体/论坛体阅读
- 语音创作与视频原生生成（MVP暂时不做）
- 商业化，包括解锁、会员订阅与广告激励（首期不纳入）

## 3. 用户画像与场景

### 3.1 目标用户
- 年龄：18-30岁
- 人群：追求新鲜体验、情绪共鸣、社交分享的Z世代
- 地域：一线及新一线城市为主

### 3.2 核心场景
- 碎片时间阅读沉浸剧情并做关键选择
- 探索同一故事不同分支结局
- 在移动端快速创作并提交审核发布
- 在Web端进行大纲树深度编辑、预览与发布管理

## 4. 信息架构

## 4.1 一级导航
- 推荐
- 广场
- 创作
- 我的

### 4.2 模块关系
- 消费侧：推荐 -> 阅读 -> 分支互动 -> 分支图探索 -> 付费解锁（引入商业化后）
- 创作侧：自由模式创作 -> 编辑优化 -> 提交审核 -> 发布分发
- Web创作侧：作品广场 -> 快捷对话创作 -> 大纲树编辑/预览 -> 发布配置 -> 发布分发
- 基础设施：审核、账号、支付、埋点、推荐策略

## 5. 用户故事与核心流程

### 5.1 用户故事（阅读侧）
- 作为新用户，我希望快速进入一个有代入感的故事并立即开始阅读。
- 作为阅读用户，我希望每个关键节点都有可选项并即时看到剧情反馈。
- 作为自由模式作品读者，我希望能输入自定义行动推动剧情。
- 作为探索型用户，我希望查看剧情分支图并回跳重选。
- 作为付费用户，我希望章节解锁过程清晰且不破坏阅读连续性。

### 5.2 用户故事（创作侧）
- 作为轻创作者，我希望在手机端快速输入设定并生成草稿。
- 作为创作者，我希望清楚看到审核状态与驳回原因。
- 作为创作者，我希望审核通过后获得推荐和广场曝光。
- 作为PC创作者，我希望通过自然语言对话与文件上传快速生成分支故事草稿。
- 作为进阶创作者，我希望在无限画布上编辑剧情节点和分支节点并实时预览PC/手机效果。

### 5.3 核心流程（阅读）
首页推荐 -> 进入作品 -> 章节阅读 -> 节点选择/输入 -> 剧情推进 -> 分支图回看/回跳 -> 继续阅读 -> 章节解锁 -> 完读/收藏。

### 5.4 核心流程（创作）
创作页输入设定 -> AI生成草稿 -> 编辑优化 -> 提交审核 -> 审核中 -> 审核通过可见/驳回返修 -> 再次提交。

### 5.5 核心流程（Web创作）
Web首页广场进入作品或进入创作页 -> 快捷对话输入/新建作品 -> 进入编辑工作台（对话区 + 大纲树 + 预览） -> 节点编排与交互配置 -> 发布配置（封面/标题/简介/标签/角色） -> 提交发布。

## 6. 功能清单（优先级）

### 6.1 P0（MVP必须）
- 推荐流：全屏故事卡、互动按钮、进入阅读入口
- 广场：推荐banner、最近阅读、分类浏览、搜索-基础榜单
- 阅读器：全屏沉浸、段落动画、显隐导航
- 分支决策：每节点3-4个预置选项，选择后即时推进
- 自定义输入分支：仅自由模式、固定结局模式作品开放
- 剧情分支图：路径展示 + 回跳重选
- AI立绘：按段落生成 + 失败兜底
- 快捷创作：移动端自由模式生成草稿
- Web首页广场：作品卡浏览 + 一键进入游玩
- Web创作页：快捷对话框 + 我的作品管理（聚合移动端与PC端作品）
- Web编辑工作台：左侧智能体对话区（可收起）、右侧大纲树编辑区 + 双端预览区
- 无限画布大纲树：剧情节点（图/文/视频）与分支节点（可选交互样式）
- 发布配置面板：封面、标题、简介、标签、角色信息必填校验
- 提交审核：先发后审，状态可追踪
- 章节解锁支付：按章付费
- 基础埋点：阅读、分支、回跳、付费、创作漏斗

### 6.2 P1（MVP后）
- 分支图按角色线/结局类型筛选
- 草稿版本对比与回溯
- 审核状态细粒度分层（机审/人审）
- 推荐策略个性化增强

## 7. 详细功能与交互设计

## 7.1 首页/推荐（V1.0功能）
### 页面目标
让用户在最短路径进入内容，提升首读转化。

### 页面结构
- 顶部：banner（活动/专题）
- 主体：竖滑全屏故事卡
- 卡片信息：高光片段、关键选择、标题、标签、热度、预计阅读时长
- 互动：点赞、收藏、评论

### 交互规则
- 上下滑：切换作品卡
- 点击卡片：进入作品详情或直接续读
- 长按卡片：不感兴趣（用于推荐降权）

### 体验要求
- 首屏1秒内展示核心信息
- 用户2次点击内进入正文

## 7.2 广场（V1.0功能）
### 页面结构
- 推荐banner：横板滑动推荐位，显示故事卡片、进入按钮    
- 继续阅读：继续上次故事阅读，显示故事卡片、进入按钮    
- 分类入口：悬疑、恋爱、奇幻等
- 搜索-榜单入口：热度榜、角色榜
- 瀑布流：作品卡片列表

### 交互规则
- 支持筛选与排序
- 支持“最近阅读”快速续读
- 支持点击作品，进入作品详情页

## 7.3 阅读器（V1.0功能）
### 页面结构
- 默认全屏阅读区
- 顶部弱化导航栏（目录、分支图、设置）
- 底部节点选项区（出现于分支节点）

### 通用交互
- 单击中部：显示/隐藏工具栏
- 段落切换：触发段落动画与背景过渡
- 阅读设置：字体、字号、亮度、背景主题

### AI立绘（按段落生成）
- 当前段进入即发起立绘请求
- 后台预加载下一段立绘
- 若生成失败：
  - 使用默认背景图
  - 提示“背景加载中，不影响阅读”
  - 不中断正文阅读流程

## 7.4 剧情分支图（V1.0功能）
### 功能目标
帮助用户理解剧情路径并探索多结局。

### 可视化元素
- 节点状态：已达成、当前、未探索
- 连线状态：主线、支线

### 回跳流程
- 用户点击历史节点
- 弹窗确认：“将从该节点重选并生成新路径，原路径保留”
- 确认后创建新路径版本并切换阅读进度

### 边界规则
- 仅可回跳已解锁历史节点
- 回跳不覆盖旧路径，保留并行路径记录
- 已解锁付费章节不重复收费

## 7.5 分支节点交互
### 轻量交互形式清单（AI需可生成）
以下交互都以“低学习成本、单手可完成、3秒内完成”为设计原则，适合嵌入文本阅读流程。

1) 选项选择（基础）
- 形式：2-4个选项按钮
- 场景：分支决策、态度选择、行动路径选择
- 复杂度：低

2) 连点推进（Tap Burst）
- 形式：在限定时间内连续点击按钮N次（如3次、5次）
- 场景：追逐、破门、心跳加速等紧张段落
- 反馈：进度条实时增长，失败走保底分支
- 复杂度：低

3) 方向滑动（Swipe Choice）
- 形式：左滑/右滑或上滑/下滑对应不同决策
- 场景：快速二选一、态度立场切换
- 反馈：滑动方向高亮提示，松手即提交
- 复杂度：低

4) 长按蓄力（Press Hold）
- 形式：长按1-2秒完成“蓄力条”
- 场景：屏息、潜行、稳定情绪等慢节奏沉浸场景
- 反馈：蓄力完成震动提示（可选），中断则失败
- 复杂度：低

5) 关键词输入（Keyword Unlock）
- 形式：输入1个关键词或短句（2-10字）
- 场景：口令、暗号、解谜线索
- 反馈：实时校验格式，提交后给出命中/偏离提示
- 复杂度：中低

6) 顺序选择（Sequence Pick）
- 形式：给出3个动作，按正确顺序点选
- 场景：拆弹、开锁、仪式步骤等轻解谜
- 反馈：每步即时反馈，对错可见，允许一次重试
- 复杂度：中低

7) 倒计时决策（Timed Choice）
- 形式：在倒计时内完成选项或动作，超时自动进入默认分支
- 场景：危机抉择、即时反应
- 反馈：剩余时间颜色渐变，超时明确提示
- 复杂度：低

### 交互组件使用规则（MVP）
- 单章节使用基础选项+最多另外2种交互，避免疲劳和打断阅读。
- 任一交互时长建议控制在3-8秒内。
- 失败必须有“可读可继续”的保底剧情，不做死路阻断。
- 所有交互都需支持弱网降级：超时后自动转预置选项分支。
- 自定义输入相关交互仅在自由模式作品开放。

### 预置基础选项（全作品）
- 每节点提供3-4个选项按钮
- 点击后按钮进入禁用态，展示“剧情推进中”
- 成功后进入下一段并展示剧情反馈

### 自定义输入（仅自由模式作品）
- 在选项区展示“自定义行动”入口
- 输入限制：2-30字
- 提交前做敏感词校验
- 提交后不可撤销（当前节点）

### AI回包展示建议
- 行动理解摘要（1行）
- 剧情结果正文
- 影响提示（可选：关系/风险变化）



## 7.6 首页创作模块（移动端快捷创作）
### 目标
在移动端完成“模式选择 -> 灵感输入/生成 -> 故事生成 -> 连续阅读 -> 发布编辑”的创作闭环；降低新手创作者上手成本，同时保留进阶模式能力。

### 页面与模式定义
1) 创作首页（默认：自由模式）
- 默认进入“自由模式”。
- 必填输入：灵感设定、题材选择、玩法模板（冒险/恋爱）。
- 支持“试试灵感盲盒”，可自动生成灵感设定并自动勾选题材。
- 点击“题材选择-更多”后，展开题材卡片列表（多选）。

2) 模式下拉选择框
- 点击顶部模式切换（向下按钮）展开选项框。
- 选项：自由模式 / 固定结局模型 / 大纲树模式。
- 切换后保持当前已填写内容（可复用字段尽量复用）。

3) 固定结局模型页
- 必填输入：灵感设定、题材选择、结局目标。
- 结局目标定义：当前故事结束时必须达成的结果（示例：“可设置多结局目标，例：
结局1-做上皇帝
结局2-让所有角色存活”）。

4) 大纲树模式提示弹框
- 在移动端选择大纲树模式后，不进入完整编辑器。
- 弹出提示：“请前往pc端创作”。

5) 故事阅读页（生成后）
- 任一模式点击“生成故事”后，跳转至故事阅读页。
- 阅读内容基于当前输入条件实时生成（模式、灵感、题材、模板/结局目标）。
- 创作者可持续向下阅读，不强制立即发布。

6) 发布编辑页（阅读页退出后）
- 从阅读界面退出时，进入发布编辑页。
- 页面头部：左上返回按钮 + 标题“发布编辑”。
- 页面提示文案：发布内容仅含设定及第一章，后续剧情由玩家自由探索，个人游玩路径不会被发布。
- 核心编辑区（按顺序）：
  - 封面区：支持“重新生成”与“相册上传”两种方式修改封面。
  - 标题（必填）：输入框 + 字数计数（如 `x/20`）。
  - 简介（必填）：多行输入框 + 字数计数（如 `x/200`）。
  - 故事标签：展示已选标签，支持“添加标签”继续自定义补充。
  - 发布权限：
    - 允许他人二创（开关）
    - 公开发布（开关）
- 底部操作区：
  - 左按钮“取消”（次级）
  - 右按钮“确认”（主CTA）
- 左上返回：回到阅读界面，不丢失当前阅读进度。
- 点击“取消”：弹出二次确认框，提示“是否退出发布，当前内容将自动存入私密作品”。
- 因网络问题退出阅读页，已设定内容和第一章自动存入私密作品。

### 信息结构与导航
- 入口：底部导航“创作”。
- 主路径（自由模式）：创作首页（自由模式） -> 生成故事 -> 阅读页 -> 发布编辑页。
- 主路径（固定结局模型）：模式切换 -> 固定结局模型页 -> 生成故事 -> 阅读页 -> 发布编辑页。
- 分流路径（大纲树模式）：模式切换 -> 大纲树模式提示弹框 -> 引导至PC端。
- 回流路径：发布编辑页左上角返回 -> 阅读页；阅读页再次退出 -> 发布编辑页。

### 详细交互规则
1) 自由模式输入规则
- 灵感设定字数建议10-80字，实时显示计数与校验状态。
- 题材支持多选；默认展示常用题材，点击“更多”展开完整题材卡片。
- 玩法模板二选一（冒险/恋爱），默认保留上次选择。
- 盲盒触发后自动回填灵感文本并预选题材，用户仍可手动改写。

2) 模式切换规则
- 下拉框默认高亮当前模式。
- 自由模式 -> 固定结局模型：玩法模板改为“结局目标”，必填项，未填写时“生成故事”置灰。
- 切换到大纲树模式：直接弹提示层，不进入移动端编辑态。
- 模式切换不清空已输入文本；如字段不兼容，保留在草稿扩展字段中。

3) 生成故事与阅读规则
- 点击“生成故事”进入生成中态，按钮防重复点击。
- 生成成功后跳转阅读页；阅读页支持连续下滑阅读。
- 生成失败时给出重试和返回入口，保留当前输入。

4) 发布编辑与退出规则
- 阅读退出默认进入发布编辑页。
- 字段校验规则：
  - 标题、简介为必填；为空时“确认”不可提交，并提示缺失项。
  - 标题超长或简介超长时给出字数超限提示，不允许提交。
  - 标题、简介初始为AI生成内容；
- 封面逻辑：
  - 封面初始为AI生成内容；
  - 支持 AI 重新生成与本地上传；
  - 封面生成/上传失败时保留原封面并提示重试。
- 标签逻辑：
  - 初始为AI生成内容；
  - 支持新增与移除标签；
  - 至少保留 1 个标签（建议），为空时提示补充。
- 权限逻辑：
  - “公开发布”关闭时，提交后进入私密作品；
  - “公开发布”开启时，提交后进入公开发布流程（先发后审）。
- 点击“确认”：
  - 提交当前封面、标题、简介、标签、权限配置；
  - 成功后进入发布成功反馈并回写至“我的作品”。
  - 成功后跳转到页面“创作”。
- 发布编辑页“取消”必须二次确认。
- 确认取消后：自动保存至私密作品并退出发布流程，跳转到页面“创作”。

### 状态机（创作首页与阅读发布链路）
自由模式编辑中 / 固定结局编辑中 -> 生成中 -> 阅读中 -> 发布编辑中 -> 已发布  
生成中 -> 生成失败 -> 重新生成/返回编辑 
发布编辑中 -> 取消确认 -> 草稿已保存  
任意编辑中 -> 切换大纲树模式 -> PC端引导态

### 异常与兜底
- AI生成失败：保留当前输入与模式，提示“重试生成”。
- 网络中断：本地自动保存输入快照，恢复后提示继续。
- 草稿保存失败：提示“保存失败，请重试”，并保留当前编辑态。
- 发布取消确认弹框异常关闭：不执行退出，不丢数据。


## 7.7 我的作品管理
### 目标
让创作者集中管理个人作品，支持按公开/私密分类查看、可见性切换、信息编辑与作品删除；降低内容管理成本，保障隐私与安全操作。

### 页面与功能定义

1) 我的作品列表页
- 入口：个人中心 > "我的作品"标签。
- 顶部双Tab分类：**公开作品** / **私密作品**，默认展示公开作品。
- 公开作品Tab下展示所有已发布且可见的作品；私密作品Tab下展示仅自己可见的作品及草稿。
- 列表项展示：封面缩略图、标题、题材·模式标签、状态标签（已发布/🔒 私密/✏️ 草稿）、阅读量（公开作品）、更新时间。
- 右侧"⋮"按钮为操作入口。

2) 作品操作菜单（底部ActionSheet）
- 点击"⋮"后弹出底部操作菜单。
- 菜单顶部显示作品标题 + 当前状态。
- 菜单项根据作品当前状态动态展示：
  - **公开作品**：设为私密 / 信息编辑 / 删除作品 / 取消。
  - **私密作品**：设为公开 / 信息编辑 / 删除作品 / 取消。
- "设为私密"说明文案：仅自己可见，不参与推荐与广场展示。
- "设为公开"说明文案：发布到推荐与广场，所有用户可见。
- "信息编辑"说明文案：编辑标题、简介、封面、标签等。
- "删除作品"说明文案：删除后不可恢复，使用红色文字强调危险操作。

3) 可见性切换交互
- 公开 → 私密：点击"设为私密"后立即生效，作品从广场/推荐下架，Toast提示"已设为私密"。
- 私密 → 公开：点击"设为公开"后需校验是否满足发布条件（标题、简介、封面非空），不满足则跳转信息编辑页补充。满足则立即生效，Toast提示"已设为公开"。

4) 信息编辑
- 点击"信息编辑"跳转到发布编辑页（复用7.6已有的发布编辑页）。
- 编辑完成保存后返回我的作品列表，更新对应条目信息。

5) 删除作品（二次确认弹窗）
- 点击"删除作品"弹出居中确认弹窗。
- 弹窗标题："确认删除作品？"
- 弹窗说明：展示具体作品名 + "将被永久删除，包括所有章节、读者数据和历史记录。此操作不可撤销。"
- 双按钮："取消"（灰底黑字）/ "确认删除"（红底白字）。
- 确认删除后作品永久移除，Toast提示"作品已删除"。

### 信息结构与导航
- 入口：个人中心 > 我的作品。
- 主路径：我的作品列表（公开Tab） -> 点击"⋮" -> 操作菜单 -> 设为私密 / 信息编辑 / 删除。
- 切换路径：公开Tab <-> 私密Tab。
- 回流路径：信息编辑完成 -> 返回列表；删除确认 -> 返回列表。

### 详细交互规则
1) 列表交互
- 点击作品卡片主体区域（封面/标题/简介）：进入阅读页。
- 点击"⋮"按钮：弹出操作菜单。
- Tab切换：保持列表滚动位置独立，切换Tab不重置另一Tab的滚动位置。
- 列表为空时展示空态："暂无公开作品" / "暂无私密作品"。

2) 可见性切换规则
- 切换操作不可批量，仅支持单作品逐个操作。
- 切换后作品在当前Tab列表中消失（移至另一Tab）。
- 操作成功后自动收起ActionSheet。

3) 删除规则
- 删除为不可逆操作。
- 删除仅限创作者本人。
- 删除不影响其他用户已缓存的阅读进度（缓存过期后自然失效）。

### 状态机
私密草稿 -> 私密已保存 -> 公开发布  
公开发布 -> 设为私密 -> 私密已保存  
私密已保存 -> 设为公开 -> 公开发布  
任意状态 -> 删除确认 -> 已删除

### 异常与兜底
- 可见性切换失败：保留原状态，提示"操作失败，请重试"。
- 删除失败：保留作品，提示"删除失败，请稍后重试"。
- 信息编辑保存失败：保留编辑态，提示"保存失败，请重试"。
- 列表加载失败：显示错误占位 + 重试按钮。


## 7.8 Web端创作工作台（广场 + 创作 ）
### 目标
在Web端建立“进入游玩 + 快速创作 + 深度编辑 + 双端预览 + 发布配置”闭环，承接移动端草稿并提供高效率生产能力。

### 页面与模块定义（以现有初版设计 `1094:10106` 为基础）
1) Web首页（作品广场）
- 页面定位：作品发现与游玩入口。
- 主体：作品卡片流（封面、标题、标签、热度、更新状态）。
- 操作：点击卡片可直接进入游玩（阅读器）或进入详情后再游玩。
- 支持基础筛选：题材、热度、最新发布（MVP）。

2) Web创作页（快捷对话 + 我的作品）
- 页面上半区：快捷对话框（Prompt输入）。
- 对话框占位引导文案示例：
  - “描述你想要的互动故事，例如：末世经营+悬疑反转，3个关键分支”
  - “也可上传小说/视频素材，让AI帮你改写为分支互动故事”
- 输入能力：支持纯自然语言输入、支持附件上传（小说文本、视频文件）。
- 页面下半区：我的作品管理（展示当前账号所有作品，含移动端与PC端创作来源）。
- 列表支持：搜索、按状态筛选（草稿/已发布/私密）、最近编辑排序。
- 点击“新建作品”或发送对话均可进入编辑页。

3) Web编辑页（工作台）
- 布局分区：
  - 左侧：智能体对话区（默认展开，可收起）。
  - 右侧上：大纲树编辑区（无限画布）。
  - 右侧下：预览区（PC预览 + 手机预览）。
- 顶部右侧：发布按钮。
- 发布按钮点击后，打开发布配置面板：封面、标题、简介、标签、角色信息。

### 详细交互规则
1) 智能体对话区
- 支持多轮对话生成与改写（如“把第2章改成双线叙事”）。
- 支持上传附件：
  - 小说文本：抽取人物、世界观与情节主线，转为可编辑节点草稿。
  - 视频文件：抽取剧情片段/节奏建议，生成分支骨架（仅结构建议，不直接输出视频内容）。
- 若用户输入意图不明确，智能体必须先发起澄清提问，再执行生成。澄清至少覆盖：题材、主角目标、分支规模（示例：3分支/5分支）。

2) 大纲树编辑区（无限画布）
- 画布能力：拖拽平移、滚轮缩放、框选、吸附对齐、小地图导航。
- 节点类型：
  - 剧情节点：支持文字、图片、视频内容块。
  - 分支节点：支持配置交互玩法样式（选项选择、倒计时、关键词输入、滑动决策等）。
- 连接规则：
  - 每个分支节点至少连接2条后续路径。
  - 节点删除前需检查后继路径，避免产生孤立链路（提示用户自动重连或同时删除）。
- 版本能力（MVP）：支持手动保存版本快照与回退到上一个快照。

3) 预览区（双端）
- 提供PC尺寸预览（默认 1440宽内容容器）与手机尺寸预览（390宽容器）。
- 支持在同一剧情节点下同步刷新两端预览内容。
- 支持“从当前节点开始试玩”与“从开头完整试玩”。

4) 发布流程
- 点击“发布”后进入发布配置面板。
- 必填项：封面、标题、简介、标签、角色信息。
- 任一必填项缺失时，发布按钮置灰并展示缺失项提示。
- 发布成功后状态为“审核中”，并回写到“我的作品”列表。

### 信息结构与导航
- Web首页广场 -> 进入游玩。
- Web创作页 -> 快捷对话或新建作品 -> Web编辑页。
- Web编辑页 -> 发布配置面板 -> 提交发布。
- 我的作品（Web）与移动端“我的作品”共享同一作品数据源（按账号聚合）。

### 状态机
Web创作输入中 -> AI生成草稿中 -> 大纲树编辑中 -> 预览中 -> 发布配置中 -> 审核中 -> 已发布  
Web创作输入中 -> 澄清问题中 -> AI生成草稿中  
任意编辑中 -> 发布校验失败 -> 编辑中  
任意编辑中 -> 自动保存失败 -> 待重试

### 异常与兜底
- 输入语义不清：阻断直接生成，先进入“澄清问题中”。
- 文件解析失败：提示失败原因（格式不支持/体积超限/解析异常）并提供重试。
- 无限画布渲染性能下降：降级隐藏非视口节点细节，仅保留结构骨架。
- 双端预览失败：保留编辑状态，提示“预览加载失败，请重试”。
- 发布失败：保留发布面板与已填字段，不清空用户输入。

### 设计一致性约束
- 风格延续移动端主色系：浅灰、白、黑、亮黄色，Web端强调信息密度与操作效率。
- 以 `1094:10106` 为基线：保留“左对话、右编辑+预览”主框架，不改变核心心智模型。
- 关键CTA（生成、发布、确认）使用统一主按钮样式，危险操作使用红色语义色。

## 8. 状态机与异常流程

## 8.1 作品发布状态机
草稿 -> 待提交 -> 审核中 -> 已发布  
审核中 -> 驳回 -> 修改中 -> 重新审核

### 状态定义
- 草稿：仅作者可见
- 审核中：不可公开，不参与推荐
- 已发布：可被搜索、推荐和广场展示
- 驳回：不可见，需修改后重提

## 8.2 阅读异常流程
- AI剧情生成超时：返回兜底剧情并后台重试
- AI立绘失败：降级默认背景，不中断阅读
- 分支提交失败：本地暂存选择，网络恢复后重放
- 分支回跳失败：停留原路径并提示重试

## 8.3 支付异常流程
- 支付成功未到账：自动轮询 + 手动“恢复购买”
- 支付失败：保留阅读位置并支持重试
- 重复点击支付：按钮防抖 + 订单幂等
